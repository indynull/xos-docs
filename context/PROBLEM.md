# Problem

Parent: [../VISION.md](../VISION.md)

## Short version

Computers push hard work through apps and habits. You glue the tools together. Chat on top of that setup does not fix it. Many Linux installs add clutter. Themes do not help.

## What goes wrong

1. **Apps as the unit of work** — real jobs cross tools all day.  
2. **Browser as glue** — structured work buried in tabs.  
3. **Decoration as product** — attention spent on looks.  
4. **Too much default software** — packages and services you did not ask for.  
5. **Split tools** — editor, terminal, files, browser feel unrelated.  
6. **Agents on top** — no saved skills, no desktop that follows the task.  
7. **Agent as the user** — one chat process with full rights, no supervisor, no ACLs.  
8. **Demo as product** — QEMU boots and theme screenshots instead of real work on real machines.

## Better

Shorter path from goal to a result you trust. Desktop matches the kind of work. Skills stick around. Built for hard jobs, not gadget demos.
