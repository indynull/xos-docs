# How to edit these docs

1. [BRIEF.md](../BRIEF.md)  
2. [VISION.md](../VISION.md)  
3. Everything else  

Change the idea in 1–2 first. Keep detail files in sync. Do not invent a second product in a side file.

| Change | File |
|--------|------|
| Short blurb | BRIEF.md |
| One-pager | VISION.md |
| Rules | principles/ |
| Concepts | concepts/ |
| v1, security, stack, decision | product/ |
| Problem, other tracks | context/ |

Log real changes in the changelog. Prefer roles over naming people. If v1 grows, update scope and non-goals together.
